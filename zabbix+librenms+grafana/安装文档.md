# 监控栈安装文档

## 环境要求

- Docker + Docker Compose
- 至少 4GB 可用内存
- 交换机需开启 SNMP v2c（community: `global`）

## 安装 Docker

### Ubuntu

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh get-docker.sh
sudo apt update && sudo apt install -y docker-compose-plugin git
sudo usermod -aG docker $USER
newgrp docker
```

### CentOS

```bash
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin git
sudo systemctl start docker && sudo systemctl enable docker
sudo usermod -aG docker $USER
```

## 启动服务

```bash
cd zabbix+prometheus+grafana
cp .env.example .env

# 如果部署在远程服务器，把 .env 里的地址改成实际访问地址：
# SERVER_IP=服务器IP
# LIBRENMS_BASE_URL=http://服务器IP:8002

# 启动所有服务
docker compose up -d

# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f

# 检查健康状态
docker compose ps --format "table {{.Name}}\t{{.Status}}\t{{.Ports}}"
```

## 网段变更

以后内网换网段时，优先改 `.env`，不用改脚本：

```env
SNMP_COMMUNITY=global
SERVER_IP=192.168.10.10
LIBRENMS_BASE_URL=http://192.168.10.10:8002
LIBRENMS_DISCOVERY_TARGETS=192.168.10.1-100,192.168.10.254
LIBRENMS_CORE_IP=192.168.10.254
SWITCH_DISCOVERY_RANGE=192.168.10.1-100,192.168.10.254
PROMETHEUS_PING_TARGETS=192.168.10.254,192.168.10.11-16
```

例如新网段是 `10.10.20.0/24`：

```env
SERVER_IP=10.10.20.10
LIBRENMS_BASE_URL=http://10.10.20.10:8002
LIBRENMS_DISCOVERY_TARGETS=10.10.20.1-100,10.10.20.254
LIBRENMS_CORE_IP=10.10.20.254
SWITCH_DISCOVERY_RANGE=10.10.20.1-100,10.10.20.254
PROMETHEUS_PING_TARGETS=10.10.20.254,10.10.20.11-16
```

应用变更：

```bash
docker compose up -d --force-recreate librenms librenms-dispatcher librenms-scheduler librenms-config zabbix-agent zabbix-config
```

如果 Grafana 的 ICMP Ping 面板也要跟着变，修改 `PROMETHEUS_PING_TARGETS`，再执行：

```bash
docker compose up -d --force-recreate prometheus
```

## 自动配置说明

### Zabbix

`zabbix-config` 容器会自动完成：
- 修复 Agent 连接问题
- 添加主机和监控模板
- 导入 WatchGuard / Hillstone 防火墙模板
- 配置飞书告警机器人

查看配置日志：
```bash
docker compose logs zabbix-config
```

手动验证 Agent 连接：
```bash
docker exec zabbix-server zabbix_get -s zabbix-agent -k system.uname
```

### LibreNMS

`librenms-config` 容器会自动完成：
- 启动并连接 Redis / dispatcher / scheduler / rrdcached
- 设置 LibreNMS 访问地址和 RRD 权限
- 启用 dispatcher 轮询、发现、告警、服务和 ping 任务
- 创建 API Token
- 创建默认管理员（admin / admin）
- 自动发现 SNMP 设备（默认范围：`192.168.10.1-100,192.168.10.254`）
- 配置设备离线和高延迟告警规则
- SNMP Community: `global`

### Prometheus / Grafana

Prometheus 只作为 Grafana 的轻量数据源：
- 采集核心/舞台交换机 SNMP
- 采集核心/舞台交换机 ICMP
- 默认不采集主机 CPU、内存、磁盘等 node-exporter 指标
- 不依赖 Grafana 启动时联网安装 Zabbix 插件
- Prometheus、SNMP exporter、Blackbox exporter 默认只绑定服务器本机 `127.0.0.1`
- Grafana 自动加载 `Network` 文件夹下的 `SNMP Stats` 和 `Blackbox exporter - ICMP` 面板

查看配置日志：
```bash
docker compose logs prometheus
docker compose logs grafana grafana-setup
```

## 常见问题

### 服务启动慢

首次启动需要拉取镜像和初始化数据库，Zabbix 可能需要 2-3 分钟。耐心等待，用 `docker compose ps` 检查状态。

### Zabbix Web 连不上

确认 `mysql` 和 `zabbix-server` 都已 healthy：
```bash
docker compose ps mysql zabbix-server
```

### LibreNMS 设备未发现

检查 SNMP 连通性：
```bash
docker exec librenms snmpwalk -v2c -c global 192.168.10.254 sysName.0
```

### LibreNMS Validate 有红色报错

首次安装后先等 3-5 分钟，让 dispatcher 跑完一个采集周期，再刷新 Validate。常见处理：

- `Poller is not running`：检查 `docker compose ps librenms-dispatcher`，再看 `docker compose logs librenms-dispatcher`
- `Scheduler is not running`：检查 `docker compose ps librenms-scheduler`，再看 `docker compose logs librenms-scheduler`
- `You have not enabled distributed_poller`：重建 `librenms-config`，脚本会自动执行 `lnms config:set distributed_poller true`
- `Redis is unavailable`：检查 `docker compose ps librenms-redis`
- `You have not enabled rrdcached`：检查 `docker compose ps librenms-rrdcached`，然后重建 `librenms-config`
- `ServerName is set incorrectly`：把 `.env` 的 `LIBRENMS_BASE_URL` 改成浏览器实际访问地址，例如 `http://130.94.66.45:8002`
- `Mail skipped`、`Non-git install updates are manual` 是 Docker 场景下可以先忽略的提示，不影响 SNMP 采集和告警基础功能

### Zabbix 提示 Agent 不可用

更新代码后重建 agent 和配置容器：

```bash
docker compose up -d --force-recreate zabbix-agent zabbix-config
docker compose logs --tail=100 zabbix-agent zabbix-config
```

`Zabbix server` 主机会自动改成通过 `zabbix-agent:10050` 连接，通常需要再等 1-3 分钟告警恢复。

### Grafana Dashboards 是空的

Grafana 面板通过 provisioning 自动加载。更新代码后重建 Grafana 即可，不要删除 `grafana-data`：

```bash
docker compose up -d --remove-orphans --force-recreate prometheus grafana grafana-setup
docker compose logs --tail=100 grafana grafana-setup
```

正常情况下会看到 `Network` 文件夹，里面有 `SNMP Stats` 和 `Blackbox exporter - ICMP` 两个面板。

如果日志里看到旧的 `alexanderzobnin-zabbix-app` 插件安装错误，说明服务器还在跑旧代码或旧容器，先 `git pull`，再用上面的命令重建 Grafana。

### 重置所有数据

```bash
docker compose down -v
rm -rf mysql-data zabbix-server-data grafana-data librenms-db-data librenms-data librenms-rrdcached-journal
docker compose up -d
```
