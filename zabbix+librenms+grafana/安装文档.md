# 监控栈安装文档

赛事网络监控（Zabbix + LibreNMS + Prometheus + Grafana），自动发现 + 模板化告警，一次部署用一个赛事，赛后整机回收。

## 一、环境

- Linux 服务器（Ubuntu 推荐 / CentOS 也行）
- Docker + Docker Compose（compose 插件版，不是老 docker-compose）
- ≥ 4 GB 可用内存（核心交换机 / 防火墙 SNMP 完整采集时建议 8 GB）
- 监控网络可达交换机、防火墙、选手网段，且这些设备开了 SNMP v2c

## 二、首次安装 Docker

### Ubuntu

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh get-docker.sh
sudo apt update && sudo apt install -y docker-compose-plugin git
sudo usermod -aG docker $USER && newgrp docker
```

### CentOS

```bash
sudo yum install -y yum-utils git
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo systemctl enable --now docker
sudo usermod -aG docker $USER && newgrp docker
```

## 三、出差部署 checklist（每次新机都跑一遍）

### 1. 拉代码

```bash
git clone https://github.com/yikkrrtykj/monitor-autoconfig.git
cd monitor-autoconfig/zabbix+librenms+grafana
```

### 2. 写 `.env`

```bash
cp .env.example .env
vi .env
```

**必改项**：

```bash
SERVER_IP=                 # 监控服务器自身 IP（局域网内可访问）
LIBRENMS_BASE_URL=http://${SERVER_IP}:8002

# 基础设施 ping（Name:IP，逗号分隔，支持 1-10 这种范围）
CORE_SWITCH_PING=Core:192.168.10.254
DIST_SWITCH_PING=SW1:192.168.10.11,SW2:192.168.10.12
FIREWALL_PING=FW1:192.168.1.1,FW2:192.168.1.2
SERVER_PING=Server:192.168.10.10

# 防火墙 SNMP（HA 两台都写）
FIREWALL_SNMP_TARGETS=192.168.1.1,192.168.1.2

# LibreNMS 自动发现网段
LIBRENMS_DISCOVERY_TARGETS=192.168.10.1-100,192.168.10.254
LIBRENMS_CORE_IP=192.168.10.254
SWITCH_DISCOVERY_RANGE=192.168.10.1-100,192.168.10.254
```

**赛事监控用**（不接选手就留空）：

```bash
TOURNAMENT_SWITCHES=192.168.10.11,192.168.10.12   # 选手接入交换机 IP
PLAYER_SUBNETS=192.168.11.0/24                    # 选手有线网段
WIRELESS_SUBNETS=192.168.12.0/24                  # 选手无线（备用，不用就留空）
```

### 3. 起服务

```bash
docker compose up -d
docker compose ps
```

首次启动 5-8 分钟（拉镜像 + 数据库初始化 + 自动配置）。

### 4. 跑赛前自检

```bash
./pre-match-check.sh
```

输出每个 dashboard 数据流通情况、设备 ping 状态、选手注册数量、ISP 链路检测、Grafana 加载情况。绿色 = OK，红色 = 要解决。

### 5. 浏览器访问

| 服务 | 地址 | 用户 / 密码 |
|---|---|---|
| Grafana | `http://${SERVER_IP}:3000` | admin / root |
| Zabbix | `http://${SERVER_IP}:8001` | Admin / zabbix |
| LibreNMS | `http://${SERVER_IP}:8002` | admin / admin |

Grafana 进去看 **Network** 文件夹：
- Event Infrastructure（设备 ping + ISP 流量 + 丢包热力图）
- Match 5v5（2 队 × 5 人，5v5 / 王者荣耀类）
- Tournament 6 队（6 队，3 人/队 / 4 人/队 通用，三角洲类）
- Tournament 64 (2 层 / 3 层 233 / 3 层 332)（16 队 × 4 人，PUBG / 大型赛事）

## 四、选手监控接线约定

`generate-player-targets.py` 通过 SNMP walk 读交换机端口 `ifAlias`（端口描述），按 `team\d+[-_]\d+` 正则解析 `team` 和 `seat` 标签。

**约定**：交换机端口描述写成 `teamNN-MM`，编号**从下往上、从左往右**递增。

例如 64 人 2 层布局，舞台左下第 1 队的 4 个选手：
```
端口 1 ifAlias = team01-01
端口 2 ifAlias = team01-02
端口 3 ifAlias = team01-03
端口 4 ifAlias = team01-04
```

3 层布局看 dashboard 顶部说明，team 编号映射有详细图。

无线选手（备用）：把无线 AP 接入网段写到 `.env` 的 `WIRELESS_SUBNETS`，dashboard 顶部"网络"选择器切到"无线"或"全部"看。

## 五、防火墙 ISP 接口命名约定

防火墙 SNMP 抓 `ifHCInOctets` / `ifHCOutOctets` 算上下行流量。Dashboard 自动按 `ifAlias` 包含 `telecom|telcom|unicom|isp|wan` 关键词（不区分大小写）筛选 WAN 口。

**约定**：在防火墙上把 WAN 口的 description / ifAlias 改成包含 ISP 名字的字符串，如：
- `ISP-Telecom-100M`
- `Unicom-WAN-Backup`
- `WAN1-Telcom`

每条 ISP 一个独立 panel 显示上下行，最多自动平铺 4 条。

## 六、网段变更（赛后换场）

新场地换网段，改 `.env`：

```bash
SERVER_IP=10.10.20.10
CORE_SWITCH_PING=Core:10.10.20.254
DIST_SWITCH_PING=SW1:10.10.20.11,SW2:10.10.20.12
FIREWALL_PING=FW1:10.10.20.1
LIBRENMS_DISCOVERY_TARGETS=10.10.20.1-100,10.10.20.254
LIBRENMS_CORE_IP=10.10.20.254
SWITCH_DISCOVERY_RANGE=10.10.20.1-100,10.10.20.254
```

应用：

```bash
docker compose up -d --force-recreate prometheus librenms librenms-dispatcher librenms-config zabbix-config player-targets
```

Grafana 不用重启（数据源指向 prometheus 服务名，IP 变了 prometheus 自己重启会感知）。

## 七、自动配置说明

### Zabbix

`zabbix-config` 容器会：
- 修复 Agent localhost 连接问题
- 创建主机、SNMP 接口
- 导入 WatchGuard / Hillstone 防火墙模板
- 配置飞书告警机器人（用 `feishu-robot.py`，token 在 Zabbix 媒介里设）

```bash
docker compose logs zabbix-config
docker exec zabbix-server zabbix_get -s zabbix-agent -k system.uname    # 验证 agent
```

### LibreNMS

`librenms-config` 容器会：
- 启 dispatcher / scheduler / rrdcached
- 创建 admin 用户和 API Token
- 自动发现 SNMP 设备（按 `LIBRENMS_DISCOVERY_TARGETS`）
- 配置 2 条告警规则：**设备离线告警**（critical）和 **高丢包告警**（warning，丢包 > 10%）
- **不接通知渠道**——告警只在 LibreNMS UI 显示，飞书告警走 Zabbix 的 `feishu-robot.py`

### Prometheus / Grafana

Prometheus 抓：
- 4 类 infra ping（core/dist/fw/server）通过 blackbox-exporter
- 防火墙 SNMP（`if_mib` 模块，64-bit 计数器）通过 snmp-exporter
- 选手 ping（file_sd 自动同步 `player-targets` 容器的输出）

Grafana 通过 file provisioning 自动加载 `grafana-provisioning/dashboard-json/` 里的 dashboard，每 30 秒重载一次。

## 八、常见问题

### Q: 服务起不来或一直重启
```bash
docker compose ps
docker compose logs --tail=100 <service-name>
```

### Q: Zabbix Web 502 / 连不上
等 mysql + zabbix-server healthy。首次 2-3 分钟正常。

### Q: LibreNMS 显示发现 0 个设备
检查 SNMP：
```bash
docker exec librenms snmpwalk -v2c -c global 192.168.10.254 sysName.0
```
不通 = 防火墙策略 / community 错 / 设备没开 SNMP。

### Q: Grafana 选手 dashboard 全是 No data
1. `./pre-match-check.sh` 看选手 targets 注册了多少
2. 0 个 = 检查 `TOURNAMENT_SWITCHES` 配了没 + 交换机端口 ifAlias 是否按约定命名
3. 注册了但都离线 = 选手电脑 / 手机没接好

### Q: dashboard 改了 .env 后不生效
- 改 IP / community / 选手网段 → 重启 prometheus 和 player-targets
- 改 dashboard JSON → Grafana 30 秒自动 reload，或 `docker compose restart grafana`

### Q: 想换其他通知渠道（钉钉 / 邮件）
Zabbix 这条线：改 `feishu-robot.py` 改成钉钉 webhook（差不多结构）。
LibreNMS 这条线：UI 里 **Alerts → Alert Transports** 添加（默认没接，要手动配）。

### Q: 重置所有数据，从头开始
```bash
docker compose down -v
rm -rf mysql-data zabbix-server-data grafana-data librenms-db-data librenms-data librenms-rrdcached-journal prometheus-data
docker compose up -d
```

## 九、赛后清理

赛事结束、服务器要回收：
```bash
docker compose down -v        # 停服务并删数据卷
docker system prune -a        # 清镜像（如果服务器还要给别人用）
```

数据要带走的话，先备份：
```bash
tar czf monitor-backup-$(date +%Y%m%d).tar.gz \
  mysql-data/ grafana-data/ librenms-data/ librenms-db-data/ prometheus-data/ .env
```

下次新场子直接 `git clone` + `cp .env.example .env` 重头来。
