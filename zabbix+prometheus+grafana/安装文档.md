# 监控栈安装文档

## 环境要求

- Docker + Docker Compose
- 至少 4GB 可用内存
- 交换机需开启 SNMP v2c（community: `global`）

## 安装 Docker

### Ubuntu

```bash
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh get-docker.sh
sudo apt update && sudo apt install -y docker-compose-plugin git
sudo usermod -aG docker $USER
newgrp docker
```

### CentOS

```bash
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin git
sudo systemctl start docker && sudo systemctl enable docker
sudo usermod -aG docker $USER
```

## 启动服务

```bash
cd zabbix+prometheus+grafana

# 启动所有服务
docker compose up -d

# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f

# 检查健康状态
docker compose ps --format "table {{.Name}}\t{{.Status}}\t{{.Ports}}"
```

## 自动配置说明

### Zabbix

`zabbix-config` 容器会自动完成：
- 修复 Agent 连接问题
- 添加主机和监控模板
- 导入 WatchGuard / Hillstone 防火墙模板
- 配置飞书告警机器人

查看配置日志：
```bash
docker compose logs zabbix-config
```

手动验证 Agent 连接：
```bash
docker exec zabbix-server zabbix_get -s zabbix-agent -k system.uname
```

### LibreNMS

`librenms-config` 容器会自动完成：
- 创建 API Token
- 添加交换机设备（Core、SW1、Stage1-6）
- 配置设备离线和高延迟告警规则
- SNMP Community: `global`

查看配置日志：
```bash
docker compose logs librenms-config
```

## 监控的网络设备

| 设备 | IP | SNMP Community | 用途 |
|------|-----|----------------|------|
| Core | 192.168.10.254 | global | 核心交换机 |
| SW1 | 192.168.10.11 | global | 接入交换机 |
| Stage1-6 | 172.25.10.3-8 | global | 舞台交换机 |

## 常见问题

### 服务启动慢

首次启动需要拉取镜像和初始化数据库，Zabbix 可能需要 2-3 分钟。耐心等待，用 `docker compose ps` 检查状态。

### Zabbix Web 连不上

确认 `mysql` 和 `zabbix-server` 都已 healthy：
```bash
docker compose ps mysql zabbix-server
```

### LibreNMS 设备未发现

检查 SNMP 连通性：
```bash
docker exec librenms snmpwalk -v2c -c global 192.168.10.254 sysName.0
```

### 重置所有数据

```bash
docker compose down -v
rm -rf mysql-data zabbix-server-data grafana-data librenms-db-data librenms-data uptime-kuma-data
docker compose up -d
```
